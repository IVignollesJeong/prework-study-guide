HTML

  • HTML is the foundation to the house (website).
  • The head element contains information about the webpage.
  • The body element represents the visible conten shown to the user.
  • Elements use opening and closing tags to insert content into the webpage.
  • These elements include h1-6 for headers, p for paragraphs, ul/ol/li for lists
  • Many more elements exist. More information on these elements can be found here!

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • The * selector applies the styling rules to ALL elements.
  • Three ways to style using CSS: inline CSS, internal CSS style sheet, and external CSS style sheet.
  • Inline is embedded within the code line and styles just that line of code.
  • Internal is embedded within the HTML file using a style element and only apply to that page.
  • External is a separate file that is linked to the HTML code. Use this to have more control/cleaner code. (Separation of concerns)
  • CSS consists of a selector and declarations. The declarations consist of a property and a value.
  • A selector selects which element or attribute to apply the style rules (the property and value) to.
  • A property is the kind of style we want to apply (ex: color, text-align).
  • A value is the description of the property we are applying (ex: blue, justify).
  • EXTERNAL CSS: you must link the .css to the HTML to see changes.
  • linking consists of the link element (link) relationship between documents (rel="") and reference to said document (href="").
  • Use . then the class name (ex: section class="card") to style that class. (ex: .card to style the card sections in our HTML).
  • More information on CSS elements and attributes can be found here!

GIT

  • git status: checks what branch we are currently on
  • git checkout main or branch-name: moves current workspace to said branch
  • git checkout -b branch-name: creates a new branch (-b) and switches to it
  • code .: launches code in Visual Studio
  • ---The following is the process of saving our work on GitHub---
  • git add -A: adds current code from VS to the current working branch
  • git commit -m "description of change": commits current code and adds a description message (-m)
  • git pull origin main: makes sure our local branch is up to date with our base (main) branch.
  • git push origin name-of-branch: pushes your changes to the remote repo online.
  • More information on Git commands can be found here!

JavaScript

  • A VARIABLE is a named container that allows us to store data in our code.
  • CONTROL FLOW is the order in which a computer executes code in a script.
  • ARRAYS are single variables that are used to hold a group of data.
  • Data within arrays are indexed, meaning each data point is assigned a number.
  • Arrays are zero-indexed and sequential, meaning they start at zero and increase by 1 each data point (ex:0,1,2,3,...).
  • LOOPS use predictable patterns of indices to perform a task on all the items in an array.
  • An example of a loop is a for loop. The for loop uses predictable patterns of indices to perform a task on all the items in an array.
  • The for loop does this by executing a code block over and over, only being interrupted conditional statements such as if statements.
  • The for loop contains three important statements: starting point of loop (ex: var x = 0;), condition (ex: x less than shapes.length;), and iteration (ex: x++).
  • A for loop would look like this: for(var x = 0; x less than shapes.length; x++) { code block }.
  • The code block could be something like printing the data on the console (ex: console.log(shapes[x]);
  • A FUNCTION is a set of instructions that tells the computer how to perform a certain task.
  • Declare a function then follow with a unique name which will be used to call the function (ex: function helloWorld() {console.log("Hello World I am a function")}).
  • The function name is always followed by closed parentheses which will later be filled with parameters we define.
  • Within the curly brackets after the function lies the code block that will be executed when the function name is called.
  • Ex: the code block console.log("Hello World I am a function") would be executed by writing helloWorld(), which would print "Hello World I am a function" in the console.
  • More information on JavaScript can be found here!
